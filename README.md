## Overview

This repository includes selected work from my CS 300 course, where I studied data structures and algorithms through the design and development of an advising assistance program for ABCU. The purpose of the program was to help academic advisors access course information, print courses in alphanumeric order, and look up individual course titles and prerequisites.

## Project Artifacts

- **Project One:** Runtime and memory analysis of different data structures, including vectors, hash tables, and binary search trees.
- **Project Two:** Working C++ code for the advising assistance program, including file input, course storage, sorted course output, and individual course lookup.

## Reflection

The main problem I worked on throughout this course was designing and building a program that could help academic advisors at ABCU access course information more efficiently. The program needed to read course data from a file, store that information in an appropriate data structure, print a list of courses in alphanumeric order, and allow a user to search for an individual course to view its title and prerequisites.

In Project One, I focused on analyzing different data structures, including vectors, hash tables, and binary search trees, to determine which structure would best support the program requirements. In Project Two, I applied that analysis by writing working C++ code for the advising assistance program.

## Approach

I approached the problem by first thinking through how the data needed to be stored, searched, and printed. This helped me understand why data structures are important in software design. Each structure had different strengths and weaknesses.

For example, a hash table is useful for fast lookup, but it does not naturally keep data sorted. A vector is simple to use, but searching through it can become slower as the amount of data grows. A binary search tree was a strong choice for this project because it allowed the courses to be printed in order while still supporting efficient searching.

Comparing these structures helped me see that choosing the right data structure depends on what the program needs to do most often.

## Challenges and Solutions

One of the biggest roadblocks I encountered was moving from pseudocode and planning into working C++ code. It was one thing to describe the logic, but another thing to make sure the program could correctly read the file, validate the data, store the course objects, and print the correct output.

I worked through this by breaking the project into smaller parts, testing each feature separately, and improving the code as I found issues. I also paid close attention to file input, menu behavior, and how prerequisites were displayed. Testing the program with the provided course data helped me confirm that the output matched the assignment requirements.

## Growth as a Developer

This project expanded the way I approach designing software because it showed me the importance of planning before coding. Instead of jumping straight into writing a program, I first compared possible designs, thought about runtime and memory, and decided which structure made the most sense for the problem.

That process helped me understand that good software design is not only about making code work, but also about making sure the solution fits the user’s needs and can handle the required operations efficiently.

## Maintainability, Readability, and Adaptability

My work in this course improved the way I write programs that are maintainable, readable, and adaptable. I became more intentional about using clear variable names, organizing code into separate functions, adding helpful comments, and keeping each part of the program focused on one responsibility.

I also learned the value of writing code that can be tested and adjusted without having to rewrite the entire program. Overall, these projects helped me build a stronger understanding of data structures, algorithms, and how thoughtful design choices can make a program more useful and easier to maintain.
